# IAMユーザーに適用するポリシー

## 概要
このシステムをTerraformでデプロイするIAMユーザーに、以下のポリシーを適用してください。

## 対象アカウント
- **AWSアカウントID**: `YOUR_AWS_ACCOUNT_ID`
- **リージョン**: `ap-northeast-1` (東京リージョン)

## 適用するポリシー

### deployment-policy.json
Terraformでインフラを構築するために必要な権限です。

このポリシーをIAMユーザーにアタッチしてください。

## 権限の内容

### Lambda関連
- Lambda関数の作成・更新・削除
- 実行権限の設定

### API Gateway関連
- REST APIの作成・設定
- デプロイメント管理
- APIキーと使用量プランの管理

### IAM関連
- Lambda実行ロールの作成・管理
- ポリシーのアタッチ

### CloudWatch Logs関連
- ロググループの作成・管理

### Bedrock関連
- モデルの呼び出し権限

### EC2関連（読み取りのみ）
- インスタンス情報の取得

## 使用方法

1. IAMユーザーを作成
2. `deployment-policy.json` の内容でポリシーを作成
3. ポリシーをユーザーにアタッチ
4. アクセスキーを発行
5. そのアクセスキーを提供

## 注意事項
- このポリシーは構築に必要な最小限の権限のみ
- 削除系の操作も含まれています（terraform destroyのため）
- 本番環境では更に制限することを推奨
